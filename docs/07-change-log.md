# 07 变更日志

## v0.4.2 — 2026-07-16

### 需求背景

上一轮前端重构后，员工详情时间线、试用期中心跟进节点、聚合接口验证、风险查询和前端测试仍需收尾。

### 已确认规则

- 员工时间线由后端统一聚合，不由前端拼接多个业务接口。
- 试用期中心一次返回当前页员工的跟进节点进度和下一节点。
- 最新风险按 `created_at DESC, id DESC` 稳定排序；当前模型没有 `assessed_at` 字段。
- 本次不新增 `latest_risk_level` 等风险冗余字段。
- 聚合接口只有 404 才允许前端临时降级，500、超时和响应结构错误必须展示错误。

### 数据库变化

- 新增 Alembic 迁移 `add_business_indexes`，补充风险、沟通、任职和跟进任务查询索引。
- 未新增风险冗余字段，未修改状态值，未删除业务数据。

### API 变化

- `GET /api/employees/{employee_id}/timeline` 返回跟进、沟通、AI、风险、评估、转正、异动、离职和再次入职等生命周期事件。
- `GET /api/probation/employees` 返回 `followup_progress`，包含已完成节点、全部节点和下一节点。
- `GET /api/dashboard/overview`、`GET /api/work-items` 已注册并通过 OpenAPI 和实际请求验证。

### 前端变化

- 员工详情页时间线改为局部加载和局部失败，不影响员工基础信息展示。
- 试用期中心展示跟进进度、下一节点、到期日期和逾期天数。
- 工作台和工作事项聚合接口降级仅限 404。
- 待入职状态不显示无效“确认入职/开始入职/立即处理”操作。

### 测试变化

- 新增 Vitest 配置和测试脚本。
- 增加生命周期、日期和下一跟进节点纯函数测试。
- 后端 pytest、前端 Vitest 和前端构建均已通过。
- 前端测试依赖升级到 `vitest@4.1.10`，`npm audit` 已确认 0 漏洞。
- 使用内存 SQLite 构造 100 名员工、每人 20 条沟通、每条 1～3 条风险评估进行批量风险和聚合查询性能验证。

### 迁移与回滚

- 已执行 `uv run alembic upgrade head`。
- 已执行 `uv run alembic downgrade -1` 和再次 `uv run alembic upgrade head`，确认索引迁移可回滚和重放。

### 风险

- 大样本性能验证未写入业务数据库；当前四类查询均低于验收目标。

## v0.4.0 — 2026-07-16

### 需求背景

系统已完成除 AI 对接和权限认证外的大部分功能，实际仅由一名 HR 使用。继续建设登录、角色、权限点和数据范围会增加无实际收益的开发与维护成本，因此 V1 改为单用户无认证模式。

### 已确认规则

- V1 不实现登录、退出、密码、JWT、Session、角色、权限点和数据范围。
- 前端启动后直接进入首页。
- 后端业务接口不要求认证信息，不返回 401 或 403。
- 后端使用固定本地操作者记录创建、修改、确认、审核和日志。
- 现有审计和 `user_id` 类字段可以保留兼容，但不关联账号权限。
- 业务状态、幂等、并发、对象关系和逻辑删除校验必须保留。
- 应用默认仅在本机或受信任内网运行，不直接暴露公网。

### 产品变化

- 移除登录、用户管理、角色管理、权限配置和无权限页面。
- 系统定位从“授权 HR 使用”调整为“一名本地 HR 使用”。
- 任务和日志统一显示固定本地操作者。

### 数据库变化

- 不创建用户、角色、权限、会话和令牌表。
- 已有审计字段保持兼容，由后端写入固定本地操作者。
- 若已有未上线认证表或外键，需通过迁移停止使用或清理，并提供回滚。

### API 变化

- 移除认证、当前用户、用户、角色和权限接口。
- 移除认证请求头和 401/403 契约。
- 删除认证中间件、权限依赖和数据范围过滤。
- 业务校验和 AI 输入归属校验保持不变。

### 前端变化

- 移除登录页、Token Store、路由权限守卫和 401/403 拦截。
- 移除用户、角色和权限菜单。
- 页面按钮只按业务状态控制。

### AI 变化

- AI 接口不校验用户权限。
- 仍必须校验沟通记录、任职和文本版本归属关系。
- AI 密钥仍只由后端环境变量读取。

### 测试变化

- 删除未登录、角色和数据范围测试。
- 增加无登录入口、无 Token、固定操作者、防操作者伪造、无鉴权链路和部署边界测试。

### 迁移与回滚

- 删除认证代码前搜索路由、中间件、依赖、前端守卫和数据库外键。
- 认证表如已存在，先备份并确认无历史依赖。
- 回滚需要恢复对应代码、表结构和环境变量，不允许只恢复前端登录页。

### 风险

- 无应用内认证时，任何能访问服务地址的人均可操作系统。
- 必须依赖本机、内网、防火墙、VPN 或外部网关限制访问。

## v0.3.0 — 2026-07-15

### 需求背景

对 v0.2.0 文档进行一致性审查后，发现日期定义、AI 与风险流程、员工识别、试用期评估、文本历史、权限配置和任务重算等规则仍不足以直接指导实现。本版本锁定 V1 架构级规则并消除跨文档歧义。

### 已确认规则

- 入职日期记为 D0。
- 跟进日期使用自然日，周末和节假日不顺延。
- 默认试用期为 3 个自然月，目标月无同日时取月末。
- 员工数据库唯一标识为内部 ID；导入按标准化姓名匹配。
- 同名或多匹配不得自动合并，必须人工处理。
- 再次入职沿用员工档案并创建新任职记录。
- 多人确认默认 `ALL`，节点允许配置 `ANY`，任务保存模式快照。
- 风险等级为 `UNSPECIFIED`、`LOW`、`MEDIUM`、`HIGH`。
- 一次 AI 调用同时生成总结和风险建议。
- AI 校验成功后自动创建待审核风险记录。
- AI 失败时允许保存纯人工风险判断。
- 沟通文本每次修改保存完整版本。
- 转正前必须有已完成的最终试用期评估。
- 离职后未完成任务统一取消，已完成任务保留。

### 产品变化

- 将影响架构的待确认项转为已确认规则。
- 明确 `PENDING` 任职状态表示未来待入职。
- 明确试用期过程评估、最终评估和转正决策的关系。
- `.txt` 导入明确不进入 V1。

### 数据库变化

- 新增概念表 `communication_text_version`。
- `communication_record` 改为引用当前文本版本。
- `ai_summary_version` 改为引用 `source_text_version_id`，不再重复保存 `source_text_snapshot`。
- `risk_assessment` 支持 AI 自动创建和纯人工创建。
- `followup_task` 增加偏移量、触发类型等节点快照字段。
- `probation_review` 增加阶段、状态、序号和沟通关联。
- `regularization_record` 关联最终评估并支持 `APPROVED`、`EXTENDED`、`NOT_APPROVED`。
- `identity_ref` 改为含义明确的 `source_employee_key`。
- 明确 `operation_log` 不使用逻辑删除和更新字段。

### API 变化

- AI 生成接口一次返回 `summary_id` 和 `risk_assessment_id`。
- 新增沟通文本历史查询接口。
- 新增纯人工风险维护接口。
- 新增试用期评估接口。
- 转正接口校验最终评估。
- 日期修改预览明确返回各类不自动重算任务。
- 新增员工匹配冲突和最终评估缺失错误码。

### 权限变化

- “可配置”明确为角色与权限点映射。
- V1 默认不实现单用户权限覆盖。
- 增加沟通文本历史和人工风险权限点。
- 明确 HR_MEMBER 默认不能删除员工和查看操作日志。

### AI 变化

- 单次调用同时返回总结和风险。
- 结果整体校验，禁止保存半份成功数据。
- 过期 AI 风险不得直接确认。
- 风险无依据时使用 `UNSPECIFIED`，不解释为确认无风险。

### 测试变化

- 增加 D0、自然日、月末试用期、PENDING 激活测试。
- 增加同名员工冲突测试。
- 增加文本完整版本和历史权限测试。
- 增加 AI 总结与风险事务一致性测试。
- 增加最终评估、延期和未通过不自动离职测试。
- 增加入职日期重算不删除任务测试。

### 文档变化

- 文档间引用统一使用 Markdown 相对路径。
- 新增 [08-deployment.md](08-deployment.md)。
- 项目根目录新增 `.env.example`。

### 迁移与回滚

- 现有沟通文本需要回填为首个文本版本。
- 旧 AI 快照迁移到文本版本或只读保留。
- `identity_ref` 迁移前检查是否包含不应继续保存的敏感身份证数据。
- 唯一索引建立前检查重复数据。
- 所有迁移必须提供备份、正向迁移、回滚和验证脚本。

## v0.2.0 — 2026-07-14

### 需求背景

录音转写成本高，免费模型仍需要服务器资源，当前没有适合部署的服务器，因此收缩 V1 范围。

### 已确认规则

- V1 不实现 ASR 录音转写。
- 不部署本地语音模型。
- 不接入 ASR 厂商。
- AI 总结基于 HR 手工输入或粘贴的沟通文本。

### 推荐规则

- V1 同时不实现录音上传和音频存储。
- 沟通文本维护版本号和哈希。
- AI 总结保存输入文本版本、哈希和必要快照。
- 文本修改后旧 AI 结果标记过期。
- AI 默认由 HR 手工触发，避免重复费用。

### 产品变化

- 移除录音上传、播放、ASR 转写和重转写。
- 沟通页面改为文本录入。
- 保留 AI 总结、风险建议和人工审核。

### 数据库变化

- 移除概念模型中的 `recording`、`transcript_version` 和 `asr_task`。
- `communication_record` 增加文本、版本和哈希。
- `ai_summary_version` 改为引用沟通文本版本。

### API 变化

- 移除录音和 ASR 接口。
- AI 接口使用 `source_text_version`。
- 增加 AI 过期标识和文本相关错误码。

### 权限变化

- 移除录音、下载、转写和 ASR 权限。
- 增加沟通敏感文本权限建议。

### AI 变化

- 输入从转写文本改为 HR 手工沟通文本。
- AI 结果追溯到文本版本。
- AI 失败不影响手工业务。

### 测试变化

- 删除录音和 ASR 测试。
- 增加文本版本、AI 过期、无 ASR 接口和无音频依赖测试。
