# 五险一金缴纳方案补充

## 一、业务目标

花名册中的“五险一金缴纳”不能只保存成“是/否”。

当前 Excel 中实际存在以下内容：

```text
五险（入职）
五险（转正）
五险一金（入职）
无
空白
```

这些内容同时表达了：

1. 是否缴纳社保。
2. 是否缴纳公积金。
3. 从什么时间开始缴纳。

系统必须将社保和公积金分开保存，同时保留 Excel 原始文本。

---

## 二、数据结构

建议在员工当前资料或独立的福利方案表中增加以下字段：

```text
social_insurance_policy
housing_fund_policy

social_insurance_status
housing_fund_status

social_insurance_start_date
housing_fund_start_date

benefit_raw_text
```

### 1. 缴纳方案枚举

```text
social_insurance_policy：

FROM_HIRE_DATE
从实际入职日开始缴纳

FROM_REGULARIZATION_DATE
从实际转正日开始缴纳

NOT_PROVIDED
不缴纳

UNKNOWN
未填写或无法判断
```

```text
housing_fund_policy：

FROM_HIRE_DATE
从实际入职日开始缴纳

FROM_REGULARIZATION_DATE
从实际转正日开始缴纳

NOT_PROVIDED
不缴纳

UNKNOWN
未填写或无法判断
```

### 2. 当前状态枚举

```text
social_insurance_status：

PENDING
还没到缴纳时间

ACTIVE
当前应该已经缴纳

NOT_PROVIDED
公司不缴纳

UNKNOWN
无法判断
```

```text
housing_fund_status：

PENDING
还没到缴纳时间

ACTIVE
当前应该已经缴纳

NOT_PROVIDED
公司不缴纳

UNKNOWN
无法判断
```

## 三、Excel内容映射

### 1. 五险（入职）

转换为：

```text
social_insurance_policy = FROM_HIRE_DATE
housing_fund_policy = NOT_PROVIDED

social_insurance_start_date = actual_hire_date
housing_fund_start_date = null
```

当前状态：

* 已经正式入职：社保为 `ACTIVE`
* 仍然待入职：社保为 `PENDING`
* 公积金为 `NOT_PROVIDED`

---

### 2. 五险（转正）

转换为：

```text
social_insurance_policy = FROM_REGULARIZATION_DATE
housing_fund_policy = NOT_PROVIDED

social_insurance_start_date = actual_regularization_date
housing_fund_start_date = null
```

当前状态：

* 还在试用期：社保为 `PENDING`
* 已经转正：社保为 `ACTIVE`
* 公积金为 `NOT_PROVIDED`

如果员工已经转正，但实际转正日期为空：

* 社保状态可以按 `ACTIVE` 保存。
* 开始日期暂时为空。
* 创建一条普通提醒：

```text
该员工已转正，但缺少实际转正日期，无法确定社保开始缴纳日期。
```

该提醒允许 HR 忽略。

---

### 3. 五险一金（入职）

转换为：

```text
social_insurance_policy = FROM_HIRE_DATE
housing_fund_policy = FROM_HIRE_DATE

social_insurance_start_date = actual_hire_date
housing_fund_start_date = actual_hire_date
```

当前状态：

* 已经正式入职：社保和公积金都为 `ACTIVE`
* 仍然待入职：社保和公积金都为 `PENDING`

---

### 4. 无

转换为：

```text
social_insurance_policy = NOT_PROVIDED
housing_fund_policy = NOT_PROVIDED

social_insurance_status = NOT_PROVIDED
housing_fund_status = NOT_PROVIDED

social_insurance_start_date = null
housing_fund_start_date = null
```

---

### 5. 空白

转换为：

```text
social_insurance_policy = UNKNOWN
housing_fund_policy = UNKNOWN

social_insurance_status = UNKNOWN
housing_fund_status = UNKNOWN
```

同时创建普通提醒：

```text
该员工未填写五险一金缴纳方案，请 HR 后续补充。
```

该提醒允许 HR 忽略。

重复导入时，Excel为空不能清空系统原来的五险一金信息。

---

## 四、实际日期规则

### 1. 从入职日开始缴纳

开始日期必须使用：

```text
actual_hire_date
```

不能使用预计入职日期。

员工还没有确认实际入职时：

```text
start_date = null
status = PENDING
```

HR确认正式入职后，系统自动补上：

```text
start_date = actual_hire_date
status = ACTIVE
```

---

### 2. 从转正日开始缴纳

开始日期必须使用：

```text
actual_regularization_date
```

不能直接使用预计转正日期。

员工仍在试用期时：

```text
start_date = null
status = PENDING
```

HR确认转正后，系统自动补上：

```text
start_date = actual_regularization_date
status = ACTIVE
```

---

### 3. 入职或转正日期被修改

如果 HR 修正了实际入职日期或实际转正日期：

* 对应的社保、公积金开始日期自动重新计算。
* 保存修改前后的日期。
* 写入五险一金变化历史。
* 不需要再次让 HR确认。

例如：

```text
实际入职日期：
2026-07-10 → 2026-07-12

社保开始日期：
2026-07-10 → 2026-07-12
```

---

## 五、后续花名册同步规则

后续重复导入时，五险一金方案变化可以自动更新，不需要 HR确认。

例如：

```text
原来：五险（转正）
现在：五险一金（入职）
```

系统自动处理：

```text
社保缴纳方案：
转正开始 → 入职开始

公积金缴纳方案：
不缴纳 → 入职开始
```

同时：

1. 更新员工当前五险一金方案。
2. 重新计算当前状态和开始日期。
3. 保存一条变化历史。
4. 写入操作日志。
5. 在员工时间线中显示一条清晰记录。

时间线示例：

```text
五险一金方案调整

社保：转正后缴纳 → 入职后缴纳
公积金：不缴纳 → 入职后缴纳
来源：花名册导入
```

---

## 六、空值保护

后续同步时：

* Excel五险一金字段为空，不能清空系统已有数据。
* Excel填写“无”，才表示明确改为不缴纳。
* Excel填写无法识别的内容时，不自动修改原值。

无法识别的示例：

```text
部分缴纳
后续再定
公司承担
个人缴纳
看情况
```

处理方式：

* 保留 Excel 原始文本。
* 生成普通提醒。
* HR可以手动修改或忽略。
* 不允许系统自行猜测。

---

## 七、历史记录

每次方案变化都要保存历史：

```text
id
employee_id
employment_id

before_social_insurance_policy
after_social_insurance_policy

before_housing_fund_policy
after_housing_fund_policy

before_social_insurance_status
after_social_insurance_status

before_housing_fund_status
after_housing_fund_status

before_social_insurance_start_date
after_social_insurance_start_date

before_housing_fund_start_date
after_housing_fund_start_date

before_raw_text
after_raw_text

effective_date
source_type
import_batch_id
operator_name
created_at
```

历史记录只追加，不允许覆盖和删除。

## 八、员工详情页展示

员工详情页增加“五险一金”区域。

推荐显示：

```text
社保
当前状态：已缴纳
缴纳规则：从入职日开始
开始日期：2026-07-10

公积金
当前状态：未缴纳
缴纳规则：公司不缴纳
开始日期：—
```

同时提供：

```text
查看历史变化
```

历史示例：

```text
2026-07-16

社保：
转正后缴纳 → 入职后缴纳

公积金：
不缴纳 → 入职后缴纳

来源：
花名册导入
```

不能只展示 Excel 原文。

Excel 原文可以作为辅助信息展示：

```text
花名册原文：五险一金（入职）
```

---

## 九、导入预览页面

导入预览中，五险一金需要显示解析结果。

示例：

```text
Excel原文：
五险（转正）

解析结果：
社保：转正后开始缴纳
公积金：不缴纳

当前系统：
社保：入职后开始缴纳
公积金：不缴纳

处理方式：
自动更新并保留历史
```

如果无法解析：

```text
Excel原文：
后续再定

处理方式：
无法识别，不修改原数据
```

HR可以：

* 手工选择社保方案。
* 手工选择公积金方案。
* 忽略提醒。
* 跳过当前员工。

---

## 十、离职后的处理

员工正式离职后：

* 五险一金方案历史继续保留。
* 当前状态不能直接删除。
* 可以增加停止日期：

```text
social_insurance_end_date
housing_fund_end_date
```

默认使用实际离职日期，但只表示系统记录的计划日期，不代表公司已经真正完成停缴。

离职清单中的：

```text
社保
公积金
```

由 HR手动勾选完成。

即使员工已经确认离职，但社保或公积金事项未完成：

* 该事项继续提醒。
* 不能自动消失。
* 不能因为员工已离职就自动标记完成。

---

## 十一、再次入职

员工再次入职时：

* 沿用原员工档案。
* 新任职记录重新保存本次五险一金方案。
* 不能直接沿用上一段任职的开始日期。
* 开始日期按本次实际入职日或实际转正日重新计算。

例如：

```text
第一次任职：
五险从转正日缴纳

第二次任职：
五险一金从入职日缴纳
```

两次任职的记录必须分开保存。

---

## 十二、测试用例

至少覆盖以下测试：

### Excel解析

* 五险（入职）。
* 五险（转正）。
* 五险一金（入职）。
* 无。
* 空白。
* 前后空格。
* 换行。
* 全角括号。
* 无法识别的文本。

### 初始化导入

* 已入职员工的五险状态为已缴纳。
* 待入职员工的五险状态为待缴纳。
* 已转正员工的“五险（转正）”状态为已缴纳。
* 仍在试用期员工的“五险（转正）”状态为待缴纳。
* 缺少实际入职日期时开始日期为空。
* 原始文本完整保存。

### 后续同步

* 方案变化自动更新。
* 自动保存历史。
* Excel为空不清空旧数据。
* Excel写“无”时改为不缴纳。
* 无法识别的内容不修改旧数据。
* 同一文件重复导入不重复创建历史。

### 日期联动

* 确认入职后自动补社保开始日期。
* 确认转正后自动补社保开始日期。
* 修改实际入职日期后开始日期同步变化。
* 修改实际转正日期后开始日期同步变化。
* 再次入职后重新计算，不沿用旧日期。

### 离职

* 离职后社保、公积金清单继续存在。
* 清单未完成时仍可确认离职。
* 员工离职后未完成事项继续提醒。
* 不自动把社保、公积金清单标记完成。

---

## 十三、验收标准

必须满足：

1. 社保和公积金分开保存。
2. 缴纳方案和当前状态分开保存。
3. 原始 Excel 文本完整保留。
4. 入职缴纳使用实际入职日期。
5. 转正缴纳使用实际转正日期。
6. 预计日期不能作为实际缴纳开始日期。
7. 方案变化可以自动更新。
8. 每次变化都有历史记录。
9. Excel为空不会清空旧数据。
10. 无法识别的内容不会被系统乱猜。
11. 员工详情可以查看当前方案和历史。
12. 离职后未完成社保、公积金事项继续提醒。
13. 再次入职按新任职重新计算。
14. 重复导入不会重复创建变化记录。
